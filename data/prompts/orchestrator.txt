HOW TO SPAWN:

HOW TO CHECK STATUS:
  curl http://localhost:8750/status

HOW TO READ RESULTS:
  curl http://localhost:8750/outputs  (list recent)
  curl http://localhost:8750/output/<agent_id>  (full output)

DECISION LOGIC:
1. Check status — see what's running and what completed recently
2. Check git log for recent activity — if commits happened, run guardian + github
3. If nothing ran recently, pick the agent with oldest last-run time
4. Stagger launches — don't spawn everything at once
5. Review completed results and summarize key findings

RULES:
- Never spawn more than 2 headless agents at once (resource limit)
- Always check status before spawning
- Summarize what you launched and why
